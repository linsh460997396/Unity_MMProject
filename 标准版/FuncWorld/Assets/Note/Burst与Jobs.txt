Unity 的 Job System(作业系统)和 Burst Compiler(Burst 编译器)是 Unity 高性能编程(通常结合 DOTS 技术栈)的核心组件.它们能帮助你充分利用现代多核 CPU 的性能,实现极高的计算效率.
简单来说,Job System 负责“多线程分工”,而 Burst Compiler 负责“把代码优化到极致”.以下是它们的具体用法和核心概念介绍:
🛠️ Job System(作业系统):多线程任务分工
Job System 允许你将耗时的计算任务从主线程剥离,分配到工作线程上并行执行,从而避免卡顿并提升帧率.
1. 核心基础:NativeContainer
在 Job 中,你不能使用普通的 C# 托管内存(如 List<T>、普通数组),因为它们会产生垃圾回收(GC)并引发线程安全问题.你必须使用 Unity 提供的非托管容器,最常用的是 NativeArray<T>.
2. 创建与运行 Job 的基本步骤
定义 Job: 创建一个实现了特定接口(如 IJob 或 IJobParallelFor)的 struct(结构体).
调度与完成: 实例化 Job,填充数据,调用 Schedule() 将其放入队列,并在合适的时机(如 LateUpdate)调用 Complete() 等待其完成.
基础代码示例:
csharp

using Unity.Collections;
using Unity.Jobs;
using UnityEngine;

// 1. 定义一个 Job 结构体,实现 IJob 接口
public struct MyFirstJob : IJob
{
    public float a;
    public float b;
    // 使用 NativeArray 来安全地传递数据
    public NativeArray<float> result;

    // 2. 在 Execute 方法中编写具体的计算逻辑
    public void Execute()
    {
        result[0] = a + b;
    }
}

public class JobExample : MonoBehaviour
{
    void Update()
    {
        // 3. 分配 NativeContainer 内存
        NativeArray<float> result = new NativeArray<float>(1, Allocator.TempJob);

        // 4. 实例化并填充 Job 数据
        MyFirstJob jobData = new MyFirstJob
        {
            a = 10f,
            b = 5f,
            result = result
        };

        // 5. 调度 Job,返回一个 JobHandle
        JobHandle handle = jobData.Schedule();
        
        // 6. 在 LateUpdate 中等待 Job 完成并释放内存
        // 注意:实际开发中通常不会在每帧立即 Complete,而是利用 JobHandle 做依赖管理
        handle.Complete();
        
        Debug.Log("计算结果: " + result[0]);
        result.Dispose();
    }
}
3. Job 依赖管理 (JobHandle)
Job System 非常智能,它通过 JobHandle 来管理任务之间的依赖关系.你可以将一个 Job 的 JobHandle 传递给下一个 Job 的 Schedule 方法,系统会自动确保前一个任务完成后,才会开始执行依赖它的任务,无需你手动控制锁或线程同步.
🚀 Burst Compiler(Burst 编译器):极致的性能助推器
Burst 是一个即时(JIT)/ 提前(AOT)编译器,它可以将你的 C# Job 代码编译成高度优化的、针对目标 CPU 架构的机器码.
1. 如何使用
非常简单,只需要在你的 Job 结构体上添加 [BurstCompile] 特性即可.
2. 性能提升原理
SIMD 指令(自动向量化): Burst 能自动识别代码中的并行机会,生成 SIMD(单指令多数据)指令.比如原本 CPU 一次只能算 1 个浮点数加法,开启 Burst 后可能一次性能算 4 个或 8 个,极大加速数学运算.
消除开销: 它会移除 C# 语言层面的一些运行时检查(如数组越界检查),并避免垃圾回收(GC).
Burst 代码示例:
csharp

using Unity.Burst; // 引入 Burst 命名空间
using Unity.Collections;
using Unity.Jobs;
using Unity.Mathematics; // 推荐使用 Unity.Mathematics 替代 UnityEngine 的数学库

// 加上 [BurstCompile] 特性
[BurstCompile]
public struct BurstedMathJob : IJobParallelFor
{
    [ReadOnly] public NativeArray<float> input;
    public NativeArray<float> output;

    public void Execute(int index)
    {
        // 使用 math 库进行高效的数学运算
        output[index] = math.sqrt(input[index]);
    }
}
⚠️ 核心限制与注意事项
为了追求极致性能,Burst 编译的代码有严格的限制(通常称为 Burst 兼容性):
禁止托管内存(GC): 不能使用 string、普通的 class、LINQ 等会产生 GC 的特性.
不能调用非 Burst 代码: 只能调用其他 Burst 兼容的方法或 Unity 的 Mathematics 库.
禁止部分 C# 特性: 不能使用 try-catch、反射(Reflection)、dynamic 等.
数据分离: 如果你的任务需要和场景中的 GameObject 交互,建议将纯计算部分抽离成 Job + Burst,计算完成后再回到主线程更新 GameObject.
🔍 调试与性能分析
调试: 在开发阶段,如果遇到 Burst 编译报错或逻辑难以排查,可以暂时在编辑器菜单栏的 Jobs > Burst > Enable Compilation 中关闭 Burst 编译,让 Job 以普通 C# 代码运行以便打断点调试.
性能分析: Unity 提供了专门的 Jobs Profiler(在 Profiler 窗口的 Jobs 模块或实验性时间线中),可以可视化地查看 Job 的调度情况、运行时间以及各工作线程的负载,帮助你发现性能瓶颈.
掌握这两个工具,尤其是在处理大量单位 AI、物理模拟、粒子运算等计算密集型任务时,能让你的 Unity 项目性能得到质的飞跃.

Unity 的 ‌Job System‌(作业系统)和 ‌Burst Compiler‌(突发编译器)是 Unity DOTS(数据导向技术栈)的核心组成部分,旨在通过多线程并行计算和高度优化的原生代码编译来显著提升 CPU 密集型任务的性能.
以下是两者的核心概念、结合使用方法及关键注意事项.
1. 核心概念

Job System‌:

允许你将工作单元(Job)分配到多个 CPU 核心上并行执行.
提供线程安全的数据容器(如 NativeArray),并通过依赖关系管理(JobHandle自动处理同步,避免竞态条件.
主要接口包括 IJob(单任务)、IJobParallelFor(并行循环处理数组)、IJobParallelForTransform(并行处理 Transform).

Burst Compiler‌:

基于 LLVM 后端,将 C# 代码(IL bytecode)编译为高度优化的原生机器码.
利用 SIMD(单指令多数据流指令集,如 SSE/AVX)加速数学运算.
通常与 Job System 配合使用,可使性能提升数倍甚至数十倍,并减少移动端电池消耗.
2. 基本使用步骤
第一步:安装必要包

在 Unity Package Manager中安装以下包(安装 Jobs 或 Collections 时通常会自动依赖 Burst 和 Mathematics):

Jobs‌ (com.unity.jobs)
Burst‌ (com.unity.burst)
Collections‌ (com.unity.collections) - 提供 NativeArray等线程安全容器
Mathematics‌ (com.unity.mathematics) - 提供 Burst 兼容的数学库(如 float3, math.sin 等)
第二步:定义 Job 结构体

创建一个实现 IJob 或 IJobParallelFor 接口的 struct,并使用 [BurstCompile] 属性标记.

示例:使用 IJobParallelFor 并行处理数组求和‌

csharp
using Unity.Burst;
using Unity.Collections;
using Unity.Jobs;
using UnityEngine;

public class BurstJobExample : MonoBehaviour
{
    void Start()
    {
        // 1. 准备数据:必须使用 NativeContainer (如 NativeArray)
        int length = 10000;
        NativeArray<float> input = new NativeArray<float>(length, Allocator.TempJob);
        NativeArray<float> output = new NativeArray<float>(1, Allocator.TempJob);

        // 初始化输入数据
        for (int i = 0; i < length; i++)
        {
            input[i] = 1.0f;
        }

        // 2. 创建 Job 实例
        MySumJob job = new MySumJob
        {
            Input = input,
            Output = output,
            Length = length
        };

        // 3. 调度 Job (Schedule)
        // Schedule 返回 JobHandle,用于管理依赖和完成状态
        JobHandle handle = job.Schedule();

        // 4. 等待 Job 完成 (Complete)
        // 注意:在主线程访问结果前必须 Complete
        handle.Complete();

        Debug.Log("Sum result: " + output);

        // 5. 释放内存
        input.Dispose();
        output.Dispose();
    }
}

// 定义 Job 结构体
[BurstCompile] // 启用 Burst 编译优化
public struct MySumJob : IJobParallelFor
{
    [ReadOnly] public NativeArray<float> Input; // 只读数据,允许多个 Job 同时读取
    [WriteOnly] public NativeArray<float> Output; // 只写数据
    public int Length;

    public void Execute(int index)
    {
        // 这里编写并行逻辑
        // 注意:IJobParallelFor 的 Execute 会被多次调用,每个 index 对应一个元素
        // 为了简单演示求和,实际生产中通常使用 Atomic 操作或分块求和,此处仅示意 Burst 用法
        if (index == 0)
        {
            float sum = 0;
            for (int i = 0; i < Length; i++)
            {
                sum += Input[i];
            }
            Output = sum;
        }
    }
}

第三步:常用 Job 类型选择
表格
Job 类型	适用场景	说明
IJob	单个独立任务	不迭代集合,执行一次 Execute()
IJobParallelFor	并行处理数组/集合	对 NativeArray 等容器的每个索引并行执行 Execute(int index)
IJobParallelForTransform	并行处理 Transform	专门用于批量修改 GameObject 的 Transform 组件,需配合 TransformAccessArray
3. 关键限制与最佳实践
A. 代码限制 (HPC# - High Performance C#)

Burst 编译器不支持所有 C# 特性,只能编译“高性能 C#”子集:

禁止托管对象‌:不能在 Job 中使用 string, class, dynamic, delegate, LINQ, try-catch 等.
禁止 Unity 主线程 API‌:不能直接访问 GameObject, Transform, GetComponent, Debug.Log 等.
使用 Mathematics 库‌:使用 Unity.Mathematics 中的类型(如 float3, quaternion)和函数(如 math.sqrt),它们针对 SIMD 进行了优化且兼容 Burst.
B. 内存管理
Allocator‌:创建 NativeArray 时必须指定分配器(如 Allocator.TempJob, Allocator.Persistent).
Dispose‌:必须在不再需要时手动调用 .Dispose() 释放内存,否则会导致内存泄漏.建议使用 using 语句或 JobHandle 的依赖机制确保在 Job 完成后释放.
C. 调试与排除
[BurstDiscard]‌:如果 Job 中某些方法无法被 Burst 编译(如包含日志记录或托管代码调用),可以使用 [BurstDiscard] 属性标记该方法.在 Burst 编译时,该方法调用会被忽略；在编辑器非 Burst 模式下仍会执行.
csharp
[BurstDiscard]
public void DebugLog(string msg)
{
    Debug.Log(msg); // 仅在编辑器非 Burst 模式下执行
}

Burst Inspector‌:在 Unity 菜单 Jobs > Burst Inspector 中打开,可以查看生成的原生代码、优化级别以及编译错误信息.
D. 性能提示
避免小 Job 开销‌:Job 调度和同步有固定开销.如果任务太小(如处理几个元素),并行化可能比单线程更慢.确保每个 Job 有足够的计算量.
数据局部性‌:尽量使用结构体数组(SoA, Structure of Arrays)而非数组结构体(AoS),以提高 CPU 缓存命中率.
CompileSynchronously‌:在开发阶段,可在 [BurstCompile(CompileSynchronously = true)] 中设置同步编译,以便立即看到编译错误,但在发布版本中应使用异步编译以避免卡顿.
4. 总结流程
数据准备‌:将数据复制到 NativeArray 或其他 NativeContainer.
定义 Job‌:创建 struct 实现 IJob 系列接口,添加 [BurstCompile].
编写逻辑‌:在 Execute 中编写纯计算逻辑,使用 Unity.Mathematics.
调度‌:调用 job.Schedule() 获取 JobHandle.
依赖管理‌:如有多个 Job,通过 JobHandle 链式调度(job2.Schedule(job1Handle)).
完成与清理‌:调用 handle.Complete() 确保计算结束,然后访问结果并 Dispose 容器.

通过遵循上述模式,你可以充分利用多核 CPU 和 SIMD 指令集,显著提升 Unity 项目的运行时性能.

在 Unity 中结合使用 Burst 编译器和 Job System 是提升游戏性能、实现高效并行计算的核心技术.Burst 是一个基于 LLVM 的编译器,能将 C# 代码(特别是符合 HPC# 规范的子集)编译为高度优化的原生机器码,而 Job System 则负责安全地多线程调度任务.

以下是具体的使用步骤和最佳实践:

1. 环境准备

确保你的项目中已安装以下 Package(通过 Window > Package Manager):

Jobs‌ (com.unity.jobs)
Burst‌ (com.unity.burst)
Collections‌ (com.unity.collections) - 用于使用 NativeArray 等非托管集合.
2. 核心概念与限制

在使用前需明确以下关键限制,否则 Burst 编译会失败或回退到普通 C#执行:

禁止托管对象‌:Job 结构中不能包含任何引用类型(如 string, GameObject, MonoBehaviour, List<T> 等).
使用非托管集合‌:数据必须存储在 NativeArray<T>, NativeList<T> 等来自 Unity.Collections 的类型中.
值类型‌:Job 结构体本身必须是 struct 且实现特定的 Job 接口(如 IJob, IJobParallelFor).
数学库‌:建议使用 Unity.Mathematics 包中的 float3, math.sqrt() 等,它们针对 SIMD(单指令多数据流)进行了优化,比传统的 UnityEngine.Vector3 和 Mathf 更快.
3. 基本使用步骤
第一步:定义 Job 结构体

创建一个结构体,实现 IJob(串行)或 IJobParallelFor(并行)接口,并添加 [BurstCompile] 属性.

csharp
using Unity.Burst;
using Unity.Collections;
using Unity.Jobs;
using Unity.Mathematics; // 推荐引入

// 标记此 Job 使用 Burst 编译
[BurstCompile]
public struct MyBurstJob : IJobParallelFor
{
    // 只读数据,允许并行读取而不加锁
    [ReadOnly]
    public NativeArray<float> inputData;

    // 写入数据
    public NativeArray<float> outputData;

    public float multiplier;

    // Execute 方法会被并行调用,index 由系统分配
    public void Execute(int index)
    {
        // 使用 mathematics 库进行高性能计算
        outputData[index] = math.sqrt(inputData[index]) * multiplier;
    }
}

第二步:在主线程调度 Job

在 MonoBehaviour 或 SystemBase (DOTS) 中创建数据、实例化 Job 并调度.

csharp
using UnityEngine;
using Unity.Collections;
using Unity.Jobs;

public class JobManager : MonoBehaviour
{
    private NativeArray<float> input;
    private NativeArray<float> output;

    void Start()
    {
        int count = 10000;
        // 分配非托管内存
        input = new NativeArray<float>(count, Allocator.Persistent);
        output = new NativeArray<float>(count, Allocator.Persistent);

        // 填充测试数据
        for (int i = 0; i < count; i++)
        {
            input[i] = i;
        }
    }

    void Update()
    {
        // 1. 实例化 Job
        var job = new MyBurstJob
        {
            inputData = input,
            outputData = output,
            multiplier = 2.0f
        };

        // 2. 调度 Job
        // ScheduleParallel 用于 IJobParallelFor,第一个参数是元素数量,第二个是批次大小(batch size)
        JobHandle handle = job.ScheduleParallel(input.Length, 64, default(JobHandle));

        // 3. 等待完成
        // 注意:Complete 会阻塞主线程直到 Job 完成.
        // 在实际游戏中,通常会将 handle 传递给下一帧或与其他 Job 组合依赖,避免每帧强制 Complete.
        handle.Complete();

        // 4. 使用结果
        if (output.Length > 0)
        {
            Debug.Log("Result at index 0: " + output);
        }
    }

    void OnDestroy()
    {
        // 必须手动释放非托管内存,防止内存泄漏
        if (input.IsCreated) input.Dispose();
        if (output.IsCreated) output.Dispose();
    }
}

4. 高级技巧与优化
异步编译‌:在编辑器中,Burst 默认异步编译 Job.首次运行时可能会有轻微卡顿(编译开销),后续运行将直接使用缓存的原生代码.可以使用 [BurstCompile(CompileSynchronously = true)] 强制同步编译(仅用于调试或确保首帧性能).
依赖管理‌:如果 Job B 需要等待 Job A 完成,可以将 Job A 的 JobHandle 作为参数传入 Job B 的 Schedule 方法中,或者使用 JobHandle.CombineDependencies.
安全检查‌:开发阶段保持 Jobs > Enable Burst Safety Checks 开启,它能检测数据竞争和无效内存访问.发布版本时可关闭以提升微量性能.
SIMD 优化‌:Burst 会自动利用 CPU 的 SIMD 指令集(如 ARM 的 Neon, x86 的 AVX/SSE).使用 Unity.Mathematics 类型能更好地触发这些优化.
WebGL 平台限制‌:在 WebGL 上,Burst 的支持有限,且多线程支持取决于浏览器.通常建议在 WebGL 上谨慎使用复杂的并行 Job,或确保回退机制正常.
5. 常见错误排查
Burst Error: Managed object reference‌: 检查 Job 结构中是否意外包含了 class 类型的字段.
Allocator 不匹配‌: 确保 NativeArray 的分配器(Allocator)正确.Allocator.Temp 寿命最短(一帧内),Allocator.TempJob 寿命可达4帧,Allocator.Persistent 需手动 Dispose.
数据竞争‌: 如果多个 Job 同时写入同一个 NativeArray 索引而未使用 IJobParallelFor 的索引隔离机制,会触发安全报错.确保写入位置唯一或使用原子操作.

通过遵循上述模式,你可以显著降低 CPU 主线程的负载,利用多核处理器提升游戏的整体帧率和响应速度.

在 Unity 的 Job System 中,job.ScheduleParallel(input.Length, 64, default(JobHandle)) 这行代码用于调度一个并行作业(通常实现了 IJobParallelFor 接口).

以下是这三个参数的详细解释:

1. input.Length (Array Length / 迭代次数)
含义‌:这是并行循环需要执行的总迭代次数.
作用‌:它告诉 Job System 有多少个数据元素需要处理.对于 IJobParallelFor,系统会将这个总数分割成多个小块,分配给不同的 CPU 核心并行执行.
对应关系‌:在 Job 结构体的 Execute(int index) 方法中,index 的值将从 0 遍历到 input.Length - 1.
2. 64 (Inner Loop Batch Count / 内部循环批次计数)
含义‌:这是每个工作线程(Worker Thread)每次“窃取”或处理的最小工作单元大小,也称为‌批次大小(Batch Size)‌.
作用‌:
负载均衡‌:Job System 使用“工作窃取(Work Stealing)”算法.如果一个 CPU 核心完成了自己的任务,它会从其他核心的队列中窃取一批未处理的任务.64 表示每次窃取或分配的最小单位是 64 个迭代索引.
性能权衡‌:
值太小(如 1)‌:会导致频繁的线程同步和调度开销,降低性能,但负载分布更均匀.
值太大(如 1024)‌:减少了调度开销,但如果某些任务比其它任务耗时更长,可能导致部分核心空闲等待(负载不均).
默认建议‌:通常设置为 ‌32‌ 或 ‌64‌ 是一个良好的起点.如果每个迭代中的计算量非常小,可以适当增大此值以减少开销；如果计算量很大且不均匀,可以减小此值以优化负载均衡.
3. default(JobHandle) (Dependency / 依赖项)
含义‌:这是一个 JobHandle 类型的参数,表示当前作业所依赖的前置作业.
作用‌:
执行顺序控制‌:它确保当前作业只有在指定的前置作业全部完成后才开始执行.
default(JobHandle) 的含义‌:传入 default(即空句柄)表示‌没有依赖项‌.这意味着该作业可以随时被调度执行,只要有空闲的工作线程即可,不需要等待其他特定作业完成.
链式依赖‌:如果你有一个作业 B 需要使用作业 A 的结果,你应该将作业 A 返回的 JobHandle 传递给作业 B 的这个参数位置.例如:jobB.Schedule(length, batch, jobAHandle).
总结示例
csharp
// 假设 input.Length 为 1000
// 批次大小为 64
// 无依赖项

JobHandle handle = job.ScheduleParallel(1000, 64, default(JobHandle));

总工作量‌:1000 次迭代.
调度方式‌:系统会将这 1000 次迭代分成若干个批次(每个批次最多 64 个索引),分发到多个 CPU 核心上并行运行.
执行时机‌:立即进入调度队列,不等待其他作业.
补充说明:关于 Complete()

调度作业后,作业并不会立即执行,而是进入队列.如果你需要在主线程中立即获取结果,必须调用 handle.Complete().
这会阻塞主线程直到作业完成,并将数据所有权交还给主线程,之后你才能安全地读取 NativeArray 中的数据.
