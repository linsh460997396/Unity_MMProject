Unity的Odin Inspector是一款强大的编辑器扩展插件,它通过声明式特性(Attribute)让你无需编写复杂的Editor脚本,就能大幅美化Inspector面板、构建自定义编辑器窗口,甚至实现自动化资源管理.下面从安装、基础特性到自定义窗口,一步步示范它的核心用法.

1. 安装与准备

从Asset Store导入Odin Inspector后,在脚本顶部引入命名空间:

csharp
using Sirenix.OdinInspector;


如果创建编辑器窗口,还需要:

csharp
using Sirenix.OdinInspector.Editor;


所有编辑器脚本必须放在Assets/Editor文件夹下,或使用#if UNITY_EDITOR包裹,避免打包出错.

2. 基础特性:美化Inspector面板

在MonoBehaviour或ScriptableObject中直接添加特性,即可改变字段在Inspector中的显示方式.

csharp
using Sirenix.OdinInspector;
using UnityEngine;

public class PlayerConfig : MonoBehaviour
{
    [Title("基础属性")]                     // 标题
    [LabelText("角色名称")]                 // 自定义标签
    public string characterName;

    [Range(1, 100)]
    [LabelText("等级")]
    public int level = 1;

    [BoxGroup("战斗属性")]                  // 分组框
    [HorizontalGroup("战斗属性/Stats")]     // 水平布局
    [LabelText("生命值"), SuffixLabel("点")]
    public float health = 100f;

    [HorizontalGroup("战斗属性/Stats")]
    [LabelText("攻击力"), SuffixLabel("点")]
    public float attack = 20f;

    [BoxGroup("技能")]
    [ValueDropdown("GetSkillList")]         // 下拉菜单
    public string primarySkill;

    [BoxGroup("操作")]
    [Button("初始化角色", ButtonSizes.Large), GUIColor(0.4f, 0.8f, 1f)]
    public void InitCharacter()
    {
        Debug.Log($"创建角色:{characterName}");
    }

    private System.Collections.Generic.IEnumerable<string> GetSkillList()
    {
        return new[] { "火球术", "治疗术", "闪电链", "隐身" };
    }
}


效果:Inspector面板自动出现分组框、水平排列的滑块、下拉菜单和彩色按钮,无需手写一行Editor GUI代码.

3. 条件显示与交互

利用ShowIf、EnableIf等特性,根据其他字段的值动态显示或启用控件.

csharp
public class ConditionalDemo : MonoBehaviour
{
    public bool showAdvanced;

    [ShowIf("showAdvanced")]               // 仅当showAdvanced为true时显示
    [LabelText("高级选项")]
    public string advancedOption;

    [EnableIf("showAdvanced")]             // 仅当条件满足时可编辑
    public int advancedValue;
}

4. 创建自定义编辑器窗口:OdinEditorWindow

继承OdinEditorWindow,即可像写普通脚本一样,用特性声明窗口界面.

csharp
using Sirenix.OdinInspector;
using Sirenix.OdinInspector.Editor;
using UnityEditor;
using UnityEngine;

public class QuickToolWindow : OdinEditorWindow
{
    [MenuItem("Tools/快速工具")]
    public static void OpenWindow()
    {
        var window = GetWindow<QuickToolWindow>();
        window.titleContent = new GUIContent("快速工具");
        window.Show();
    }

    [FolderPath(RequireExistingPath = true)]
    [LabelText("输出路径")]
    public string outputPath = "Assets/";

    [EnumToggleButtons, BoxGroup("设置")]
    public ScaleMode scaleMode;

    [Button(ButtonSizes.Gigantic), GUIColor(0, 1, 0)]
    public void Execute()
    {
        Debug.Log($"在 {outputPath} 执行操作,模式:{scaleMode}");
    }
}


点击菜单Tools/快速工具,一个带路径选择、按钮组和巨型按钮的窗口就出现了.

5. 带导航栏的窗口:OdinMenuEditorWindow

继承OdinMenuEditorWindow并实现BuildMenuTree,可以创建左侧带导航树的工具窗口,适合管理大量配置.

csharp
using Sirenix.OdinInspector.Editor;
using UnityEditor;
using UnityEngine;

public class DataManagerWindow : OdinMenuEditorWindow
{
    [MenuItem("Tools/数据管理器")]
    public static void OpenWindow()
    {
        var window = GetWindow<DataManagerWindow>();
        window.titleContent = new GUIContent("数据管理器");
        window.Show();
    }

    protected override OdinMenuTree BuildMenuTree()
    {
        var tree = new OdinMenuTree();
        // 添加“角色”分类
        tree.Add("角色/战士", new CharacterData { className = "战士", hp = 150 });
        tree.Add("角色/法师", new CharacterData { className = "法师", hp = 80 });
        // 添加“武器”分类
        tree.Add("武器/长剑", new WeaponData { weaponName = "长剑", damage = 25 });
        return tree;
    }

    // 内嵌数据类,用于展示
    public class CharacterData
    {
        [LabelText("职业")]
        public string className;
        [LabelText("生命值")]
        public int hp;
    }

    public class WeaponData
    {
        [LabelText("武器名")]
        public string weaponName;
        [LabelText("伤害")]
        public int damage;
    }
}


打开窗口后,左侧会显示“角色”和“武器”两个可折叠菜单,点击任意项,右侧Inspector区域会展示对应的数据对象,可直接编辑.

6. 使用Visual Designer(可视化设计器)

如果你不想在代码中添加大量特性,Odin 4.0+提供了Visual Designer.在Inspector面板中点击脚本组件右上角的Odin -> Customize in Visual Designer,可以拖拽创建分组、添加特性,所有修改保存为.ovdf文件,打包时直接移除Odin插件即可,代码保持干净.

7. 进阶:批量处理与资源管理

结合Unity的AssetDatabase、Selection和System.IO,Odin能快速构建自动化工具.例如批量生成ScriptableObject:

csharp
using Sirenix.OdinInspector;
using Sirenix.OdinInspector.Editor;
using UnityEditor;
using UnityEngine;

public class BatchSOCreator : OdinEditorWindow
{
    [MenuItem("Tools/批量生成SO")]
    public static void OpenWindow() => GetWindow<BatchSOCreator>().Show();

    [LabelText("生成数量")]
    public int count = 5;

    [FolderPath]
    [LabelText("保存路径")]
    public string savePath = "Assets/Data/";

    [Button("生成")]
    public void Generate()
    {
        for (int i = 0; i < count; i++)
        {
            var so = ScriptableObject.CreateInstance<MyData>();
            so.id = i;
            so.name = $"Data_{i}";
            AssetDatabase.CreateAsset(so, $"{savePath}Data_{i}.asset");
        }
        AssetDatabase.SaveAssets();
        AssetDatabase.Refresh();
        Debug.Log($"已生成 {count} 个 ScriptableObject 到 {savePath}");
    }

    public class MyData : ScriptableObject
    {
        public int id;
    }
}

小结

Odin的核心思路是 ‌“用特性描述UI,让框架处理绘制”‌.无论是简单的字段美化、动态显示,还是复杂的编辑器窗口、资源管线,都能用极少的代码实现.建议从[Title]、[BoxGroup]、[Button]等基础特性开始,逐步尝试OdinEditorWindow和OdinMenuEditorWindow,你会很快感受到编辑器开发效率的质变.
